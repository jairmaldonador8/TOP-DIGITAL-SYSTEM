# Sincronización de campañas Meta — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use ultrapowers:subagent-driven-development (recommended) or ultrapowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Sincronizar campañas y métricas (gasto, conversaciones) de la Marketing API v25 de Meta hacia `campanias`/`campania_finanzas`, con cron diario de Vercel y botón "Sincronizar ahora", solo lectura.

**Architecture:** Módulo `src/lib/meta/` (cliente Graph API con fetch nativo + mapeo puro + orquestador de sync con service-role). Dos disparadores del mismo módulo: route handler GET `/api/cron/meta-sync` (CRON_SECRET) y server action solo-admin. Vínculo cliente↔cuenta vía `clientes.meta_ad_account_id`; upsert idempotente por `campanias.meta_campaign_id`; bitácora en `sync_runs`.

**Tech Stack:** Next.js 16.2.10 (App Router), Supabase (service role vía `createAdminClient()`), Vercel Cron, Vitest. Sin SDK de Meta: fetch nativo contra `graph.facebook.com/v25.0`.

**Skills:** @.claude/skills/meta-marketing-api/SKILL.md (endpoints, rate limits, gotchas), `supabase-multitenant-rls` (RLS), `nextjs16-supabase-auth` (proxy/auth), `ultrapowers-dev:testing-tdd`, `ultrapowers-dev:resilience` (backoff).

**Referencias:** Spec `docs/ultrapowers/specs/2026-07-22-meta-marketing-api-sync-design.md` · Research `docs/ultrapowers/research/2026-07-22-meta-sync-research-brief.md`.

**Preferencias:** auto-commit ON, auto-push ON → cada tarea termina en commit + push.

**Convención de código:** identificadores y comentarios en español, siguiendo el estilo del repo (ver `src/lib/acciones.ts`). Tests con Vitest en `__tests__/` junto al módulo. Correr tests: `npx vitest run <ruta> --reporter=verbose`.

```ts
// src/lib/meta/mapeo.ts
/**
 * Funciones puras que traducen respuestas de la Graph API al dominio.
 * Ver .claude/skills/meta-marketing-api/SKILL.md para el porque de cada regla.
 */
import type { AccionInsight } from './tipos'

export type EstadoCampania = 'activa' | 'pausada' | 'archivada'

/** Columna "Resultados" de Ads Manager para campanas de clic a WhatsApp. */
const ACCION_CONVERSACIONES =
  'onsite_conversion.messaging_conversation_started_7d'

export function estadoDesdeMeta(effectiveStatus: string): EstadoCampania {
  if (effectiveStatus === 'ACTIVE') return 'activa'
  if (effectiveStatus === 'ARCHIVED' || effectiveStatus === 'DELETED') {
    return 'archivada'
  }
  // PAUSED, CAMPAIGN_PAUSED, ADSET_PAUSED y estados intermedios
  // (IN_PROCESS, WITH_ISSUES, PENDING_REVIEW...): no estan corriendo.
  return 'pausada'
}

export function conversacionesDe(actions: AccionInsight[] | undefined): number {
  const accion = actions?.find((a) => a.action_type === ACCION_CONVERSACIONES)
  const valor = Number(accion?.value)
  return Number.isFinite(valor) ? valor : 0
}

export function gastoDe(spend: string | undefined): number {
  const valor = Number(spend)
  return Number.isFinite(valor) ? valor : 0
}
```

```ts
// src/lib/meta/client.ts
/**
 * Cliente minimo de la Graph API v25 (solo lectura). Sin SDK: fetch nativo.
 * Server-only: usa META_SYSTEM_TOKEN / META_APP_SECRET.
 */
import 'server-only'

import { createHmac } from 'node:crypto'

import type { PaginaGraph } from './tipos'

const BASE = 'https://graph.facebook.com/v25.0'
const CODIGOS_THROTTLE = new Set([80000, 80004])
const CODIGOS_LEGACY = new Set([4, 17])

export class ErrorMeta extends Error {
  constructor(
    public codigo: number,
    mensaje: string
  ) {
    super(`Meta API (${codigo}): ${mensaje}`)
  }
}

type Dormir = (ms: number) => Promise<void>
const dormirReal: Dormir = (ms) => new Promise((r) => setTimeout(r, ms))

export async function obtenerTodos<T>(
  ruta: string,
  params: Record<string, string> = {},
  dormir: Dormir = dormirReal
): Promise<T[]> {
  const token = process.env.META_SYSTEM_TOKEN
  const secret = process.env.META_APP_SECRET
  if (!token || !secret) {
    throw new Error('Faltan META_SYSTEM_TOKEN / META_APP_SECRET')
  }

  const url = new URL(`${BASE}${ruta}`)
  for (const [k, v] of Object.entries(params)) url.searchParams.set(k, v)
  url.searchParams.set('access_token', token)
  url.searchParams.set(
    'appsecret_proof',
    createHmac('sha256', secret).update(token).digest('hex')
  )

  const filas: T[] = []
  let siguiente: string | null = url.toString()
  let reintentos = 0

  while (siguiente) {
    const respuesta = await fetch(siguiente)
    const cuerpo = (await respuesta.json()) as PaginaGraph<T> & {
      error?: {
        code: number
        message: string
        estimated_time_to_regain_access?: number
      }
    }

    if (cuerpo.error) {
      const { code, message } = cuerpo.error
      const esperable =
        CODIGOS_THROTTLE.has(code) || CODIGOS_LEGACY.has(code)
      if (esperable && reintentos < 2) {
        reintentos += 1
        const minutos = cuerpo.error.estimated_time_to_regain_access ?? 0
        const ms = CODIGOS_THROTTLE.has(code)
          ? Math.max(minutos, 1) * 60_000
          : 1000 * 2 ** (reintentos - 1)
        await dormir(ms)
        continue // reintenta la misma URL
      }
      throw new ErrorMeta(code, message)
    }

    reintentos = 0
    filas.push(...(cuerpo.data ?? []))
    // Unica senal fiable de fin: que no venga paging.next (una pagina
    // puede traer data vacia y aun tener next).
    siguiente = cuerpo.paging?.next ?? null
  }

  return filas
}
```

```ts
export type ResultadoSync =
  | { ok: true; campaniasActualizadas: number; errores: string[] }
  | { ok: false; mensaje: string }

/** Dispara la misma sincronizacion que el cron, a peticion del dueno. */
export async function sincronizarAhora(): Promise<ResultadoSync> {
  if (!(await esAdmin())) {
    return { ok: false, mensaje: 'No tienes permiso para realizar esta acción' }
  }
  try {
    const resumen = await sincronizarMeta('manual')
    revalidatePath('/agencia', 'layout')
    return {
      ok: true,
      campaniasActualizadas: resumen.campaniasActualizadas,
      errores: resumen.errores.map((e) => e.mensaje),
    }
  } catch (error) {
    console.error('Error al sincronizar con Meta:', error)
    return { ok: false, mensaje: 'No se pudo sincronizar con Meta' }
  }
}
```

## Riesgos y notas

- **Token placeholder hasta la sesión con el dueño**: el sistema debe degradar con aviso visible, nunca crash (Task 4 aísla errores por cliente; Task 7 muestra toasts).
- **Idempotencia**: upsert por `meta_campaign_id` — corridas dobles del cron (posibles en Vercel) no duplican filas.
